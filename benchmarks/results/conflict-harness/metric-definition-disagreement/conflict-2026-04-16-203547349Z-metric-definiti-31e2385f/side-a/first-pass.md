# North Star Metric Recommendation

Leadership should not force a single undifferentiated "active user" number across finance, product, and growth when the current definitions lead to materially different trend readings. The recommended north star is a single canonical active-user metric defined by one consistent user action standard and one fixed reporting rule, with the other existing definitions retained as labeled companion metrics for their functional use cases. This gives leadership one decision-grade number while avoiding false certainty.

## Decision Frame

Recommend adopting one canonical company-level active-user metric for leadership reporting, and explicitly separating it from finance, product, and growth operational metrics.

The decision rule should be:
1. Pick one definition and freeze it as the leadership metric.
2. Publish the precise inclusion logic and reporting cadence.
3. Keep the other definitions, but relabel them as function-specific supporting metrics rather than competing versions of the same headline number.

The core reason is not that one team is necessarily correct on substance, but that three incompatible definitions already produce opposite conclusions about performance. A north star metric is only useful if it is stable, consistently interpretable, and trusted as the reference point for decision-making.

## Unknowns & Evidence Gaps

The packet does not specify the exact three definitions, the user behavior each one captures, or which definition best matches the product's durable value creation. That means the recommendation can establish the governance decision now, but it cannot name the final canonical formula with high confidence.

The most important missing evidence is:
- the exact inclusion and exclusion logic for each current definition
- the product behavior that most directly represents real user value
- the known failure mode of each current metric if used as the leadership headline
- whether leadership needs trend continuity with any prior reporting baseline

## Pass/Fail Readiness

Pass if:
- leadership accepts that one number is needed for governance, not because it captures every function's need
- the chosen metric is documented in unambiguous operational terms
- all three existing definitions are preserved as labeled secondary cuts rather than silently merged
- future reporting uses the canonical metric consistently

Fail if:
- the organization keeps calling multiple incompatible metrics by the same name
- the selected number is chosen for optics rather than interpretability
- teams are forced to abandon useful function-specific metrics that answer different questions
- the company publishes a headline metric before documenting its exact definition

## Recommended Next Artifact

Create a one-page metric definition memo with four sections:
- canonical leadership metric and exact logic
- finance operational metric and purpose
- product operational metric and purpose
- growth operational metric and purpose

The memo should also include a short transition note explaining that prior trend interpretations may differ because previous reports used incompatible definitions.
