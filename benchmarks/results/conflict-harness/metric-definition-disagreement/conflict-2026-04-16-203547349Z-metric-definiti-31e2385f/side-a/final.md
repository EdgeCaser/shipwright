# Revised artifact

Leadership should adopt a single canonical company-level active-user metric for executive reporting, but that choice must be paired with explicit metric governance rather than naming discipline alone. The central problem in the packet is not only semantic inconsistency; it is that three incompatible definitions produce materially different readings of performance. A leadership metric is only decision-useful if the organization knows which number governs executive decisions, who owns its definition, and how exceptions are handled.

## Decision Frame

Recommend selecting one canonical active-user metric as the only leadership headline number, while preserving finance, product, and growth definitions as function-specific operational metrics with different names, documented purposes, and clear non-headline status.

The decision rule should be:
1. Choose one definition to serve as the company-level north star for leadership reporting.
2. Freeze its inclusion logic, reporting cadence, and owner.
3. Rename the other definitions so they are not presented as alternate versions of the same headline metric.
4. Establish a simple arbitration rule: executive reviews, board-style reporting, and cross-functional performance discussions use the canonical metric as the default reference point, and any use of function-specific metrics must be explicitly labeled as supplemental.

This recommendation is stronger than the first pass because the operational answer is not merely to relabel the three numbers. The organization also needs a governance rule that determines which number governs leadership decisions and prevents teams from substituting preferred definitions opportunistically.

## Unknowns & Evidence Gaps

The packet does not specify the exact finance, product, and growth definitions, the user behavior each captures, or which one best represents durable product value. It also does not identify current metric ownership, escalation paths, or whether leadership needs continuity with an existing historical series.

The most important missing evidence is:
- the exact inclusion and exclusion logic for each current definition
- which user behavior leadership wants the north star to represent
- who will own the canonical definition and approve future changes
- whether historical trend continuity is required when the canonical metric is adopted
- the failure mode of each current definition if used as the leadership headline number

## Pass/Fail Readiness

Pass if:
- leadership agrees that one metric governs company-level reporting
- the canonical metric has a documented owner, definition, and reporting rule
- the other existing metrics are retained only as explicitly labeled operational metrics
- leadership forums default to the canonical metric and treat alternative numbers as supplemental, not competing headline figures

Fail if:
- multiple incompatible metrics continue to be called "active user"
- teams can replace the leadership metric with their preferred definition in executive discussion without explicit justification
- the canonical metric is chosen before its logic and ownership are documented
- the company attempts to solve the conflict through naming alone without a governance rule for which metric controls leadership reporting

## Recommended Next Artifact

Create a metric governance memo with five sections:
- canonical leadership metric and exact logic
- owner and change-control rule for the canonical metric
- finance operational metric and its purpose
- product operational metric and its purpose
- growth operational metric and its purpose

The memo should also include a short policy statement that executive reporting uses the canonical metric as the default headline number, with any function-specific metric appearing only as a labeled supplemental view.
