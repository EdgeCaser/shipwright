# Revised artifact

A single permanent north star should not be declared from the current three definitions until leadership first ratifies what counts as delivered value and what entity is being counted. The right recommendation is a staged one: choose a **value-realization metric class** as the future north star, but treat the next reporting cycle as an explicit reconciliation period rather than pretending one of the three existing numbers already resolves the dispute.

## Recommendation
Recommend **Primary Value Active Accounts/Users** as the intended north star class: unique entities that complete the product's agreed primary value event within a fixed reporting window. However, because the value event and counting entity are still unresolved, leadership should **not** anoint one of the current three definitions as the durable north star today. For the immediate leadership ask, publish a single provisional headline number only if it is clearly labeled **Interim Active Metric**, paired with the other two definitions and a decision deadline to replace it with the ratified value metric.

This is not avoidance. It is governance discipline. When existing definitions differ by 40% and imply opposite growth narratives, forcing one undefined or politically convenient number into permanent north-star status creates false precision and weakens trust in the metric system.

## Operating Rule
Use this sequence:
1. Select the metric class now: value realization, not finance recognition or reach.
2. Ratify within a short fixed window the primary value event, counted entity, window, and exclusions.
3. Backfill history under that final definition.
4. Replace the interim headline with the ratified north star.

## Why This Is The Best Recommendation
A north star must be decision-useful, stable, and tied to customer value creation. Finance-oriented definitions are useful for monetization and reporting discipline. Growth-oriented definitions are useful for acquisition and engagement sensitivity. Neither should automatically win the north-star role unless it also captures realized customer value. The current dispute is therefore not just semantic; it shows the company has not yet defined the business outcome the north star is supposed to represent.

The correct executive move is to resolve the metric design question explicitly rather than hide it behind a premature single number.

## Decision Frame
The decision is whether to force one of the current incompatible active-user definitions into permanent north-star status, or to define the north star correctly before canonizing it. Recommendation: choose a value-realization metric class as the final north star direction, but treat any immediate single number as interim until the primary value event and counted entity are formally ratified.

## Pass/Fail Readiness
Pass if:
- Leadership accepts that the permanent north star must be defined by delivered value, not departmental ownership.
- A primary value event can be agreed unambiguously.
- The counted entity can be fixed as user or account based on how value is actually realized.
- Historical backfill can be produced before the metric is used for trend claims.
- Any immediate headline number is explicitly labeled interim and time-bounded.

Fail if:
- Leadership insists on a permanent single number before agreeing the value event and counted entity.
- The interim number is presented as definitive rather than provisional.
- The chosen metric can be materially inflated by low-intent activity or accounting artifacts.
- The organization refuses to preserve companion finance and growth metrics after ratification.
