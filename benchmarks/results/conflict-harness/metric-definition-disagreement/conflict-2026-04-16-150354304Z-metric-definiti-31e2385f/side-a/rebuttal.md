- Target Side: side_b
- Target Claim IDs: side_b-claim-1
- Attack Type: evidence_gap
- Severity: high

## Claim Under Attack
The leadership north star should be a value-realization metric, with finance and growth definitions retained as companion metrics rather than collapsed into one overloaded active-user number.

## Evidence Or Reason
Side B asserts Core Active Users is the correct north star because it measures 'value realization,' but the artifact itself acknowledges the primary value event is unspecified and the correct counting entity (user vs. account) is unresolved. This means Side B is recommending a metric whose definition is structurally incomplete. A north star whose core operationalization is TBD does not resolve the leadership dispute—it defers it. Leadership asked for a single number now; Side B's answer is effectively 'define the primary value event first,' which is a precondition, not a recommendation. Furthermore, the 40% divergence across existing definitions is attributed to departments 'compressing different questions into one label,' but no evidence is offered that the three current definitions actually map cleanly to value/finance/growth buckets—that mapping is asserted without support. If the growth team's definition is closer to value realization than Side B assumes, the entire governance split collapses.
